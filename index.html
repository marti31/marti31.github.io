INTRODUCCION
TaronjaBank es una aplicación web realizada en el segundo curso del ciclo formativo de DAW y que intenta simular como seria de una manera poco compleja la gestión de un banco desde el punto de vista de un trabajador y un cliente (parte no implementada aún). Para introducir la explicación de los detalles de la aplicación y de las posibles mejoras que se podrían implementar me gustaria hacer referencia a la metodologia de trabajo y a las tecnologías más importantes empleadas para la realización de este proyecto.

ESTRUCTURA DEL CODIGO FUENTE

El paquete common es un cajón de sastre donde está aquello que se va ha utilizar en toda la aplicación, como la clase de Validaciones y JsonTransformer, una herramienta para convertir JSON a objetos JAVA y viceversa.

En el paquete servicio están recogidos todos los servicios adicionales de la aplicación, en este caso el servicio de Seguridad.

En el paquete dominio están recojidas todas las clases de negocio de la aplicación, al igual que las clases Credencial y Transacción que aunque no pertenezcan 100% al negocio de la aplicación son de mucha importancia para diferentes funcionalidades de Taronja-Bank

ESTRUCTURA DEL CODIGO FUENTE

El paquete persistencia esta directamente relacionado con la base de datos, en el estan las clases relacionadas con esta y su implementación mediante Hibernate, así como las clases BussinessException y BussinessMessage que nos permiten generar los errores lanzados al realizar alguna acción erronea en la base de datos.
